# Multi Custom Agent Service - 테스트 시나리오

## TC-001: 권한 테스트 (403 응답)
**목적:** 사용자별 모드 권한 확인
**조건:**
- user-001: chatbot-c 접근 불가 설정됨
**단계:**
1. chatbot-c에 Tool 모드로 접근
2. 403 응답 확인

## TC-002: 기존 API 하위호환
**목적:** `/api/chat` 엔드포인트 정상 작동
**단계:**
1. mode 파라미터 없이 호출
2. 챗봇의 default_mode로 실행되는지 확인

## TC-003: 세션 메모리 유지
**목적:** Agent 모드에서 대화 맥락 유지
**단계:**
1. "내 이름은 홍길동이야" 입력
2. "내 이름이 뭐야?" 입력
3. "홍길동" 포함 응답 확인

## TC-004: Tool 모드 메모리 없음
**목적:** Tool 모드는 히스토리 저장 안 함
**단계:**
1. Tool 모드로 첫 메시지 전송
2. 동일 세션에서 두 번째 메시지
3. history API로 조회 시 빈 배열 확인

## TC-005: mode 오버라이드
**목적:** 요청 시 mode로 default_mode 오버라이드
**단계:**
1. default_mode=agent인 챗봇을 tool로 호출
2. ToolExecutor로 실행되는지 확인

## TC-006: 다중 DB RAG 검색
**목적:** 여러 DB에서 검색 및 병합
**단계:**
1. chatbot-a (db_001, db_002) 호출
2. 두 DB에서 검색된 결과 확인

## TC-007: 에러 처리
**목적:** 잘못된 요청에 대한 적절한 에러 반환
**단계:**
1. 존재하지 않는 chatbot_id 호출
2. 404 응답 확인
3. 잘못된 mode 값 전달
4. 400 응답 확인
